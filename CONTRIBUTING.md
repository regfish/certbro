# Contributing to certbro

`certbro` is published as open source by regfish.

Forks, bug reports, compatibility reports, and discussions are welcome.

At this time, external pull requests are not accepted for this repository.
Please use GitHub Issues and GitHub Discussions to report problems, request
features, and share integration feedback.

## How to Contribute

Use one of these channels:

- `Issues` for reproducible bugs and documentation problems
- `Issues` for compatibility reports with Linux distributions, `nginx`,
  `apache`, `caddy`, and related environments
- `Discussions` for questions, ideas, deployment patterns, and general product
  feedback

## Before You Open an Issue

Please include enough context to make the report actionable:

- `certbro version`
- Linux distribution and version
- Web server and version, if relevant
- Exact command that was executed
- Relevant logs or CLI output
- Whether the problem affects `issue`, `renew`, `reissue`, `import`, or
  `install`

For TLS and DNS provisioning issues, include the affected behavior, but do not
post API keys, private keys, certificate private material, or other secrets.

## Pull Requests

This repository currently accepts pull requests from collaborators only.

If you built a useful improvement in a fork, please open an issue or discussion
first and describe the change. That keeps review and roadmap decisions in one
place without requiring direct upstream code contributions.

## Security

Please do not report security issues in public GitHub issues or discussions.

See [SECURITY.md](SECURITY.md) for the responsible disclosure process.
